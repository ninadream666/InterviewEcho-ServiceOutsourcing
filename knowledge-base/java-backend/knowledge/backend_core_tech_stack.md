# Java后端岗位核心技术栈知识库

本文档基于一线大厂及典型后端面试真实重点考频整理，涵盖了Java后端工程师所必须掌握的核心技术领域。

## 1. Java基础 (JavaSE)
- **集合框架**：掌握List、Set、Map等主要集合类的底层数据结构及扩容原理。
- **并发与容错**：深入理解Error和Exception的继承体系及深浅拷贝差异。
- **反射与泛型**：熟悉类型擦除原理，以及Java在运行时动态获取对象属性的反射机制。
- **动态代理**：掌握基于JDK（实现接口）或CGLIB（继承类）生成代理类的方法，它是AOP的基石。
- **String特性**：理解底层不可变性的设计考量及String/StringBuilder/StringBuffer的区别。

## 2. 多线程与并发编程 (JUC)
- **线程池规范**：掌握核心参数配置、线程执行流转与四大拒绝策略。
- **锁机制底层**：理解Synchronized锁升级过程及ReentrantLock基于AQS底层的状态流转。
- **CAS与原子类**：理解无锁乐观并发修改机制以及底层Unsafe包作用。
- **ThreadLocal机制**：掌握线程本地隔离存储原理机制，及可能导致的内存泄漏场景。
- **JMM内存模型**：理解Java工作内存与主内存通信及volatile关键字如何保证可见性。
- **策略模式应用**：理解ThreadPoolExecutor拒绝策略等场景中，基于接口与具体实现类分离的策略模式设计。

## 3. JVM虚拟机层 (JVM)
- **运行时数据区**：理解内存结构（如堆、栈、方法区等区域）的具体作用。
- **类加载子系统**：掌握双亲委派模型的原理以及Tomcat等打破双亲委派的设计初衷。
- **编译与解释机制**：掌握JIT即时编译技术以及Java代码到机器码转化过程中的热点探测。
- **对象的全生命周期**：清楚对象并不是全部在堆上分配，结合对象分配漏斗理解逃逸分析机制。
- **垃圾回收 (GC)**：熟悉根可达性分析、三色标记法原理以及常见垃圾回收器各自的优劣。
- **故障排查调优**：具备线上OOM问题排查与JVM内存泄漏排查的分析思路。
- **线上Dump操作**：掌握通过jmap命令或JVM参数（如HeapDumpOnOutOfMemoryError）在产生OOM时生成堆快照文件。

## 4. 数据库技术 (MySQL)
- **存储引擎结构**：掌握InnoDB与MyISAM的区别以及B+树索引数据结构的选型合理性。
- **事务与MVCC机制**：理解ACID特性、RC（读已提交）与RR（可重复读）的差异，及MVCC版本链的控制。
- **核心日志系统**：掌握RedoLog、UndoLog、BinLog分别在数据恢复与主从复制中的作用。
- **索引体系深入**：区分聚簇索引和非聚簇索引，深入理解回表查询及覆盖索引等优化策略。
- **SQL性能优化**：具备通过Explain查看执行计划并判断与避免索引失效的优化能力。
- **数据分片与路由**：理解分库分表的垂直、水平拆分策略，并掌握防范数据倾斜的分片键(Sharding Key)选择逻辑。

## 5. 缓存层 (Redis)
- **数据结构与场景**：熟悉五大基础数据类型的底层实现及具体业务场景。
- **缓存高可用设计**：掌握如何防范与治理缓存击穿、缓存穿透和缓存雪崩问题。
- **高并发防坑**：处理Redis热点Key、大Key产生的影响导致线上慢查询及阻塞阻断思路。
- **分布式锁原理**：理解Redis为什么能做分布式锁及其应对超时、续期（看门狗机制）的保障。
- **内存管理机制**：掌握LRU、LFU等缓存淘汰策略及过期Key删除机制（定期+惰性策略）。
- **数据持久化系统**：对比RDB(全量快照)和AOF(增量日志)持久化方式，及混合持久化结构。
- **高可用与主从同步**：掌握Redis从节点增量同步和全量同步（RDB文件）避免单点故障的机制。
- **单线程与网络模型**：解释Redis即使是单线程仍为何极快（基于内存及I/O多路复用），并了解6.0多线程网络IO。
- **本地缓存并发控制**：掌握通过多线程访问控制及双重检查锁等方式，防止本地高开销数据加载过程中的缓存穿透与资源浪费。

## 6. 消息队列 (以Kafka为主)
- **三大痛点治理**：深入解决消息队列最关心的不丢失、防重复（幂等补偿）及顺序消费问题。
- **基础核心概念**：掌握ISR同步副本队列及高吞吐的原理。
- **Rebalance机制**：关注消费者组发生Rebalance的触发条件、弊端及避免策略。
- **高并发积压排查**：能够给出出现海量消息堆积导致业务降级或宕机时的紧急扩容排障策略。

## 7. 主流大厂微服务框架 (Spring 生态)
- **IOC与AOP核心**：掌握控制反转、解决循环依赖(三级缓存)原理及面向切面编程。
- **Bean生命周期**：掌握Spring Bean的作用域及从实例化到销毁全流转机制。
- **声明式事务**：理解Spring事务的传播行为及可能导致@Transactional注解失效的真实场景。
- **SpringBoot原理**：掌握自动化装配的核心实现及Starter自定义起步机制。
- **Spring MVC流程**：熟悉从DispatcherServlet接收请求到HandlerMapping寻址及最后视图与数据返回的完整流转。
- **依赖注入机制**：清晰辨别@Autowired（按类型）与@Resource（按名称）的底层区别及其适用作用域。

## 8. 分布式与架构基础
- **分布式系统基石**：理解CAP不可能三角及BASE保证最终一致性的权衡理论。
- **核心组件设计**：了解系统负载均衡的机制以及RPC微服务调用链是如何设计的。
- **系统一致性设计**：掌握在复杂架构下系统数据一致性双写（如Redis与MySQL）的处理范式。
- **稳定性防腐层**：具有与外部合作方系统对接时的隔离、限流与防拖垮降级思维。
- **BASE理论模型**：理解基本可用、软状态与最终一致性的含义及其在追求高可用分布式架构中的指导意义。
- **一致性哈希算法**：掌握将节点和数据分布在哈希环状结构上的实现逻辑，以及防范数据倾斜（虚拟节点）与容忍动态扩缩容的机制。

## 9. 计算机网络
- **HTTP协议体系**：清楚HTTP及HTTPS安全加密核心，熟悉常用的HTTP状态码。
- **TCP底层流转**：了解TCP稳健建立连接防脏数据（三次握手）及断开回收(四次挥手&2MSL)的原理。
- **粘包与拆包**：理解TCP流式传输带来的现象及处理边界。
- **网络资源分发**：梳理CDN就近节点访问逻辑及键入网址到屏幕显示的全链路。
- **TCP连接队列**：理解Linux内核维护的半连接队列（SYN）和全连接队列（Accept）在高并发场景下的流转机制。
- **DNS解析过程**：掌握从本地缓存、递归查询到迭代查询最终获取目标IP的完整域名解析链路。
- **SYN攻击防范**：理解利用半连接队列阻塞导致的拒绝服务攻击原理，及调整队列大小、开启防范机制相关的内核参数的应对策略。

## 10. 操作系统基础
- **内核资源隔离**：清晰辨别用户态与内核态在不同权限级别执行及上下文切换的开销。
- **进程与线程调度**：理解OS视角下的进程通信、协程结构及孤儿/僵尸进程。
- **IO多路复用**：深度剖析select、poll与epoll技术在高并发服务中如何承载万级连接。
- **死锁以及虚拟内存**：掌握进程发生死锁的4个必要条件，以及虚拟内存机制的设计目的。
- **常用Linux指令**：熟练掌握grep、tail、find、kill和系统状态查看等常用命令行操作以支持线上问题排查。
- **系统中断机制**：理解发生中断时CPU捕获信号进行核切换上下文的过程及硬件与软件中断的区别。

## 11. 经典场景题与算法 (方案设计)
- **外部系统防腐与限流**：面对第三方不可靠接口时，如何在当前系统网关或客户端层面做好防拖垮与限流（如令牌桶）。
- **海量数据Top K问题**：遇到千万级或1亿数据筛选时，熟练借助最小堆(Min Heap)等结构实现局部极值寻找。
- **一致性结构设计**：掌握如何结合Canal与MQ等保证复杂分布式环境下Redis与MySQL的数据双写一致。
- **面试手撕算法思路**：熟练表达常用快排等树、栈、图解法结构，具备将动态规划快速应用到实际问题中的抽象能力。
- **QPS评估与瓶颈**：能够基于接口平均耗时、服务器节点数及配置等因素合理估算系统QPS，并识别可能在数据库层的性能瓶颈。
- **博弈论与常考智力题**：深入理解“丢鸡蛋”和“取石子（尼姆博弈）”等经典逻辑题，掌握动态规划和局势必胜策略等分析解法。
